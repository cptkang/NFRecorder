# 통합 요구사항 정의서: 무장애 달성 전광판

## 1. 시스템 개요
본 시스템은 IT 서비스의 연속 가동 시간을 시각화하여 운영 팀의 동기부여와 신뢰도를 높이는 것을 목적으로 한다. 관리자가 설정한 기점으로부터 현재까지의 시간을 실시간으로 추적하며, 장애 발생 시 관리자가 수동으로 리셋하여 기록을 관리한다.

## 2. 기능 요구사항 (Functional Requirements)

### 2.1 대시보드 화면 (Public View)
- **실시간 무장애 카운터**: 관리자가 설정한 '장애 복구 완료 일시'부터 현재 시각까지의 경과 시간(일, 시, 분, 초)을 실시간으로 표시한다.
- **목표 달성 프로그레스**: 설정된 목표 일수 대비 현재 진행률을 시각적 바(Bar)와 퍼센트(%)로 출력한다.
- **기록 관리**: 역대 최장 무장애 달성 기록(Best Record)을 상시 게시한다.
- **상태 인디케이터**: 현재 서비스 상태를 관리자가 설정한 값(정상/주의/장애)에 따라 색상별로 표시한다.
- **최근 장애 이력**: 직전에 발생했던 장애의 종료 일시와 간단한 사유를 하단에 출력한다.

### 2.2 관리자 페이지 (Admin Console)
- **시간 설정 (Time Management)**:
    - **카운트 시작점 설정**: 마지막 장애 복구 일시를 캘린더 및 시간 선택기(Picker)를 통해 설정한다.
    - **목표 시간 설정**: 달성하고자 하는 목표 무장애 일수를 입력한다.
- **상태 제어 (Status Control)**:
    - 현재 서비스 상태(Normal, Warning, Critical)를 수동으로 전환한다.
    - 'Critical(장애)' 선택 시, 대시보드 카운터는 즉시 멈추고 화면은 적색 테마로 변경된다.
- **기록 리셋 및 저장**:
    - **Reset 버튼**: 장애 발생 시 클릭하여 현재까지의 기록을 히스토리에 저장하고 카운터를 0으로 초기화할 준비를 한다.
    - **역대 최고 기록 갱신**: 현재 기록이 기존 최고 기록을 초과할 경우 자동으로 업데이트하거나 관리자가 수동 승인한다.
- **텍스트 관리**: 대시보드 상단에 표시될 서비스 명칭과 공지 메시지를 수정한다.

## 3. UI/UX 요구사항 (Design Requirements)

### 3.1 비주얼 스타일
- **고대비 레이아웃**: 검은색 배경(#0A0A0A)에 형광 그린 또는 사이언 계열의 텍스트를 사용하여 관제 센터의 시인성을 확보한다.
- **폰트**: 숫자 데이터는 디지털 감성을 주는 JetBrains Mono 또는 세리프가 없는 굵은 고딕체를 사용하여 멀리서도 식별 가능하게 한다.
- **반응형 설계**: 대형 TV 화면(Full HD/4K)에 최적화하되, 관리자 페이지는 태블릿이나 PC 환경에서 조작하기 편리한 구조로 설계한다.

## 4. 데이터 요구사항 (Data Requirements)
- **영속성 유지**: 브라우저를 새로고침하거나 서버가 재시작되어도 관리자가 설정한 시작 시간과 최고 기록 데이터는 유지되어야 한다. (Internal Database 또는 LocalStorage 활용)
- **오차 방지**: 클라이언트(브라우저) 시간이 아닌 서버 시간을 기준으로 경과 시간을 계산하여 기기별 시간 차이를 방지한다.

## 5. 레이아웃 상세 설계 (Draft)

### [Main Dashboard]
- **Top**: 서비스 명칭 (텍스트) | 현재 상태 (LED 인디케이터)
- **Center**: 무장애 누적 일수 (대형 숫자: 000 DAYS 00:00:00)
- **Middle**: 목표 달성률 게이지 바 (Progress Bar)
- **Bottom**: [최고 기록] | [목표 일수] | [직전 장애 종료일]

### [Admin Page]
- **입력창**: 시작 일시 설정 (Input Type="datetime-local")
- **입력창**: 목표 일수 설정 (Number Input)
- **라디오 버튼**: 상태 선택 (Stable / Warning / Critical)
- **버튼**: 기록 초기화(Reset) 및 저장
